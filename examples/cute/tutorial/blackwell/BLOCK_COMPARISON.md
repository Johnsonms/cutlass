# Block (0,0) vs Block (0,2) 对比分析

基于程序输出，Thread 0 在不同 block 中的数据对比：

## 相同的部分

### 全局张量（所有 block 共享）
```
mA: gmem_ptr[16b](0x7f1ef4e00000) o (512,256):(256,_1)
mB: gmem_ptr[16b](0x7f1ef4e40000) o (1024,256):(256,_1)
mC: gmem_ptr[32b](0x7f1ef5000000) o (512,1024):(1024,_1)
mD: gmem_ptr[32b](0x7f1f21000000) o (512,1024):(1024,_1)
```
**结论：** 所有 block 看到相同的全局内存地址

### MMA Tiler（固定配置）
```
mma_tiler: (_128,_256,_64)
```
**结论：** 所有 block 使用相同的 tile 大小

---

## 不同的部分 ⭐

### 1. MMA 坐标
```
Block (0,0): mma_coord = (0,0,_)
Block (0,2): mma_coord = (0,2,_)
              ↑相同  ↑不同(N维度)
```
**含义：**
- 第1维(V): 都是 0 - 同一个 peer CTA
- 第2维(M): 都是 0 - 处理相同的 M 范围
- **第3维(N): 不同！** - Block (0,2) 处理第 2 个 N tile

### 2. gA - Matrix A 的 tile（相同）
```
Block (0,0): gmem_ptr(0x7f1ef4e00000) o (_128,_64,4):(256,_1,_64)
Block (0,2): gmem_ptr(0x7f1ef4e00000) o (_128,_64,4):(256,_1,_64)
```
**为什么相同？**
- 两个 block 的 M 坐标相同 (都是 0)
- 都处理相同的 128 行数据
- 起始地址相同

### 3. gB - Matrix B 的 tile（不同！）⭐
```
Block (0,0): gmem_ptr(0x7f1ef4e40000) o (_256,_64,4):(256,_1,_64)
Block (0,2): gmem_ptr(0x7f1ef4e80000) o (_256,_64,4):(256,_1,_64)
                      ^^^^^^^^
                      地址不同！
```

**地址偏移分析：**
```
Block (0,2) 地址 = Block (0,0) 地址 + offset
0x...e80000 = 0x...e40000 + 0x40000

0x40000 = 262144 字节
       = 131072 个 half_t 元素 (每个 2 字节)
       = 512 列 (256 × 2)

验证：Block (0,2) 的 N 坐标 = 2
      每个 tile N 维度 = 256
      所以 offset = 2 × 256 × sizeof(half_t) = 512 × 2 = 1024 字节
```

**等等，计算不对！让我重新算：**
```
Matrix B 布局：(1024,256):(256,_1)  K-major
每行 256 个元素 × 2 字节 = 512 字节

Block (0,2) 在 N 维度偏移 = 2 × 256 = 512 列
但是 B 是 K-major，列偏移不是连续的！

实际上：
N 维度每个 tile = 256
stride = 256 (每移动 1 列，需要跳 256 个元素)
offset = 256 × 256 = 65536 个元素
       = 65536 × 2 字节 = 131072 = 0x20000

但实际看到是 0x40000 = 2 × 0x20000
这是因为 N 坐标 = 2，所以 2 × 0x20000 = 0x40000
```

### 4. gC - Matrix C 的 tile（不同！）⭐
```
Block (0,0): gmem_ptr(0x7f1ef5000000) o (_128,_256):(1024,_1)
Block (0,2): gmem_ptr(0x7f1ef5000800) o (_128,_256):(1024,_1)
                      ^^^^^^^^
                      地址不同！
```

**地址偏移分析：**
```
offset = 0x800 = 2048 字节
       = 512 个 float 元素 (每个 4 字节)

Matrix C 布局：(512,1024):(1024,_1)  N-major (row-major)
每行 1024 个元素

Block (0,2) 在 N 维度偏移 = 2 × 256 = 512 列
offset = 512 × sizeof(float) = 512 × 4 = 2048 字节 = 0x800  ✓
```

### 5. gD - Matrix D 的 tile（不同！）⭐
```
Block (0,0): gmem_ptr(0x7f1f21000000) o (_128,_256):(1024,_1)
Block (0,2): gmem_ptr(0x7f1f21000800) o (_128,_256):(1024,_1)
                      ^^^^^^^^
                      地址不同！
```

**同 gC，offset = 0x800 = 2048 字节**

## 总结

### Block (0,0) 和 Block (0,2) 的工作分配：

| 属性 | Block (0,0) | Block (0,2) | 说明 |
|------|------------|------------|------|
| **M 坐标** | 0 | 0 | 处理相同的行范围 [0:128) |
| **N 坐标** | 0 | 2 | 处理不同的列范围 |
| **gA** | 0x...e00000 | 0x...e00000 | ✓ 相同（M相同） |
| **gB** | 0x...e40000 | 0x...e80000 | ✗ 不同（N不同）|
| **gC/gD** | 0x...000000 | 0x...000800 | ✗ 不同（N不同）|

### 关键理解：

1. **相同 M 坐标** → 处理矩阵 A 的相同行
2. **不同 N 坐标** → 处理矩阵 B/C/D 的不同列
3. **GEMM 计算：**
   ```
   Block (0,0): C[0:128, 0:256]   = A[0:128, :] × B[0:256, :]
   Block (0,2): C[0:128, 512:768] = A[0:128, :] × B[512:768, :]
                              ↑ 相同行，不同列
   ```

4. **地址偏移公式：**
   - Matrix B (K-major): `offset = N_coord × tile_N × stride_N × sizeof(type)`
   - Matrix C (N-major): `offset = N_coord × tile_N × sizeof(type)`
